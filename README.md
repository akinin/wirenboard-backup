# Wiren Board Backup

Резервное копирование Wiren Board на NFS штатными скриптами контроллера.

## Возможности

- архив конфигураций через `download_configs.sh`;
- полный архив через `download_everything.sh`;
- независимая периодичность `Ежедневно`, `Еженедельно` или `Ежемесячно` для каждого типа;
- независимое расписание и ротация для каждого типа;
- MQTT-статус и управление;
- автоматическое MQTT Discovery для Home Assistant;
- виртуальное устройство `wb_backup` в интерфейсе Wiren Board через `wb-rules`;
- защита от одновременного запуска двух архивов;
- атомарная запись: незавершённый архив имеет суффикс `.partial`.

## Установка одной командой

Выполните на Wiren Board под `root`:

```sh
bash -c "$(curl -fsSL https://raw.githubusercontent.com/akinin/wirenboard-backup/main/install.sh)"
```

Приложение устанавливается в `/opt/wirenboard-backup`. Настройки и состояние хранятся отдельно в `/etc/wb-backup` и `/var/lib/wb-backup`.

## Настройки по умолчанию

- NFS: `10.10.100.11:/volume1/Backups`;
- каталог: `WirenBoard/<hostname>`;
- конфигурации: ежедневно в `03:30`, хранить 14;
- полный архив: по воскресеньям в `04:30`, хранить 4.

Настройки находятся в `/etc/wb-backup/config.json`. После ручного изменения:

```sh
wb-backup-ctl apply
```

## Управление

```sh
wb-backup-ctl run-config
wb-backup-ctl run-full
wb-backup-ctl status
```

В Home Assistant создаются кнопки запуска, расписание, настройки ротации, статус, даты последних архивов и датчик ошибки.

В интерфейсе Wiren Board появляются два самостоятельных устройства:

- **Бэкап конфигураций** — состояние, последний запуск, файл, результат, кнопка запуска и собственное расписание;
- **Полный бэкап** — состояние, последний запуск, файл, результат, кнопка запуска и собственное расписание.

Для каждого типа можно выбрать `Ежедневно`, `Еженедельно` или `Ежемесячно`. В еженедельном режиме день задаётся словом, например `Воскресенье`; в ежемесячном — числом от `1` до `28`. Время задаётся одной строкой `ЧЧ:ММ`. Отдельных ползунков часа и минуты нет.

## Home Assistant

Для Home Assistant рекомендуется интеграция [Wiren Board Discovery](https://github.com/akinin/wirenboard-discovery). Отдельная стандартная MQTT-интеграция и MQTT bridge не нужны.

1. Установите `Wiren Board Discovery` через HACS и перезапустите Home Assistant.
2. Откройте **Настройки → Устройства и службы → Добавить интеграцию → Wiren Board Discovery**.
3. Укажите адрес Wiren Board, порт `1883` и параметры MQTT-доступа контроллера.
4. В мастере выбора отметьте элементы устройств **Бэкап конфигураций** и **Полный бэкап**.

Интеграция читает стандартные топики Wiren Board `/devices/...`. Скрипт не публикует отдельные `homeassistant/...` MQTT Discovery-сущности, поэтому дубликатов не возникает. YAML-сенсоры не требуются.

Для панели можно добавить карточку «Сущности» и выбрать:

- общий статус;
- время, файл и результат последнего архива конфигураций;
- время, файл и результат последнего полного архива;
- две кнопки ручного запуска;
- отдельное включение и периодичность каждого расписания;
- время запуска строкой `ЧЧ:ММ`, день запуска и количество хранимых копий.

День недели полного бэкапа можно указать по-русски или числом: `Понедельник`…`Воскресенье`, либо `1`…`7`.

Если интеграция была настроена до установки бэкапов, откройте её настройки и добавьте новые элементы. При необходимости используйте повторное обнаружение или перезагрузите интеграцию.

## Удаление

```sh
curl -fsSL https://raw.githubusercontent.com/akinin/wirenboard-backup/main/uninstall.sh | sh
```
